# Tasks

This directory is reserved for complex investigations that need a durable work log.

Create `docs/tasks/{task-name}/` only when the work is larger than a routine fix or straightforward feature. Each task should include:

- `README.md` for current status, problem, goal, and outcome.
- `changelog.md` as an append-only investigation diary with newest entries first.
- `task-description.md` for the original request or problem statement.
- Optional analysis, implementation-plan, experiments, archive, or sources files when they are useful.

There are no active documented tasks at the moment.
